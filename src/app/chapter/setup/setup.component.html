CLI

The Angular CLI is a crucial tool in setting up your Angular projects. You can also read and use the quickstart. Make sure you have node.js and npm installed. Otherwise you won't be able to install and use the Angular CLI.

Install the Angular CLI using: 

npm install -g @angular/cli
Creating a project

You can create an Angular project via the command line. Navigate to the folder in which you want to setup your Angular project and type: 

ng new <ProjectName>

Important: Don't forget to navigate to your new project folder before generating components!

You can fire up your app with the following command and checking out the live result via http://localhost:4200.

ng serve
Using bootstrap

You can use (for instance) Bootstrap to style your application. You need to install it into your Angular application. First navigate to your Angular project folder and then type in the following to install the latest version of Bootstrap. 

npm install bootstrap@latest

Bootstrap is now installed into your project. However, you still need to tell Angular to load the CSS. Go to your angular.json in your project root folder. Look up the styles array and add the Bootstrap CSS so that the styles configuration will look like this: 

"styles": [
    "./node_modules/bootstrap/dist/css/bootstrap.min.css",
    "src/styles.css"
]
Project structure

The configuration and initialization files (angular.json, index.html, ...) are located in the root folder. All modules (Angular, Bootstrap, ...) are located in node_modules. All code is located in the src folder. Click the image to enlarge.

How your app is loaded

First, the webserver loads the index.html file like with any other normal website (1). The Angular system is loaded in the index.html file and will trigger main.ts file to be loaded (2). Inside the main.ts file the AppModule from app.module.ts is loaded (3). The AppModule loads the initial component, the AppComponent in app.component.ts (4). Finally, in this component, like in all other components, the selector is declared (5) and so Angular can now render the right template (HTML) file.
