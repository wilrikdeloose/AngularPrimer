Introduction

Components are the main building blocks of every Angular application. One of the first things you need to do when creating a new app is determine what will be and what will not be a component.

Create components manually

There are a few steps in creating new components.

  1. Create a new folder for the component, for instance test. This can be a sub-folder of anther component if it will be used inside that component. All components are basically childeren of the App component.
  2. Create the files you need inside the test folder:
    • test.component.ts, obligatory for any component since it will define the component
    • test.component.html, the optional, but recommended template file
    • test.component.css, optional style-sheet for the component
    • test.component.spec, optional testing file
  3. Define the component in the test.component.ts file
  4. Register the component with the Angular app in the AppModule: import the test.component.ts file and declare the TestComponent

To manually create a component, you need to go through the following two steps.

  1. In the component file, export a class. Give the class an @Component decorator with a JavaScript object as paramater. Each component has two obligatory parts.
    1. Declare the selector: selector: 'app-test'
    2. Declare a template, either inline or via an external file and declaring the URL to that file
  2. Add TestComponent to the list of Declarations in the AppModule and import the test.component.ts file.

From hereon the test.component.ts file is what you call the component. The HTML file is called the template of the component.

Create components via the CLI

Except for the actual programming, all of the above steps can be done via the CLI using the following command:

ng generate component <ComponentName> --spec false

Setting the spec flag to false skips the creation of a test file. The above command can be abbreviated with:

ng g c <ComponentName> --spec false

Both commands are the same and create the component, template and style sheet as well as register the component with the AppModule.

Example

Below you'll find a TestComponent (which can also be found in this primer) with an inline template and style. A component with an inline template and styles may look like this:

import { Component } from "@angular/core";

@Component({
    selector: 'app-test',
    template: '<p>It works!</p>',
    styles: ['p { color: red; }']
})
export class TestComponent { }

Instead of the template you can point to a template file using templateUrl. For the styles, you can define styleUrls using an array or URL's pointing to several style sheets.

Notes

A component is, next to being an Angular component, just a class. It can have (private or public) attributes and methods. Those attributes can be used via the component. The template can access all public attributes and methods directly as we will see shortly.
